Exercícios Fáceis

  • 1. Crie um container Docker com a imagem do Nginx e execute-o em segundo plano.
  • 2. Crie um Dockerfile simples para construir uma imagem com Node.js.
  • 3. Inicie um container MySQL com uma senha de root configurada via variáveis de ambiente.
  • 4. Liste todos os containers em execução usando `docker ps`.
  • 5. Crie e remova uma imagem Docker usando o comando `docker rmi`.
  • 6. Execute um container Python e rode um script básico dentro dele.
  • 7. Mapeie a porta 80 do container para a porta 8080 do host.
  • 8. Use o comando `docker logs` para ver os logs de um container em execução.
  • 9. Crie um volume Docker e use-o para armazenar dados persistentes de um container.
  • 10. Use o comando `docker exec` para acessar um terminal dentro de um container em execução.
  • 11. Configure uma rede Docker personalizada e conecte dois containers a ela.
  • 12. Inicie um container Redis e teste a conexão com `redis-cli`.
  • 13. Crie um container Docker com um arquivo de configuração personalizado mapeado por um volume.
  • 14. Faça o build de uma imagem Docker usando `docker build` e uma imagem Dockerfile local.
  • 15. Suba um container com um servidor de aplicação web simples em PHP.
  • 16. Adicione uma variável de ambiente em um container do Docker com o comando `-e`.
  • 17. Crie um arquivo `.dockerignore` para ignorar arquivos durante o build.
  • 18. Use o comando `docker stop` para parar um container em execução.
  • 19. Inicie um container em modo interativo com `-it`.
  • 20. Crie e use uma imagem Docker com múltiplas etapas para otimizar o build.

Exercícios Médios

  • 1. Crie um arquivo `docker-compose.yml` para configurar um ambiente com Nginx e PHP-FPM.
  • 2. Configure um serviço multi-container usando Docker Compose para rodar uma aplicação web com backend e banco de dados.
  • 3. Crie um script de inicialização do Docker Compose para rodar containers em sequência.
  • 4. Configure um container para rodar um script de backup automático diário.
  • 5. Use o `docker swarm` para iniciar um cluster de containers em modo de replicação.
  • 6. Crie um arquivo `docker-compose.override.yml` para usar durante o desenvolvimento local.
  • 7. Configure múltiplas redes de containers em um arquivo Compose e defina a comunicação entre eles.
  • 8. Utilize um serviço de monitoramento como Prometheus em um container Docker.
  • 9. Configure um container para usar a imagem oficial do Elasticsearch com o volume persistente.
  • 10. Crie um ambiente de desenvolvimento com `docker-compose` para uma aplicação com Node.js e Redis.
  • 11. Configure um serviço Docker Compose para executar um cron job em um container.
  • 12. Adicione uma política de reinicialização de container em um arquivo `docker-compose.yml`.
  • 13. Configure o `docker-compose.yml` para mapear volumes para persistência de dados em MongoDB.
  • 14. Use `docker exec` para rodar comandos de teste em um container em execução.
  • 15. Configure um serviço de logs centralizados usando Fluentd e Docker.
  • 16. Crie um container para execução de uma aplicação Python Flask e conecte-o a um banco PostgreSQL.
  • 17. Configure variáveis de ambiente sensíveis de forma segura no Docker Compose.
  • 18. Use `docker-compose scale` para escalar a aplicação e rodar múltiplas réplicas de um serviço.
  • 19. Configure uma rede externa e faça o roteamento de tráfego para um container usando `docker-compose`.
  • 20. Configure um container para rodar um servidor Jenkins em Docker para integração contínua.
