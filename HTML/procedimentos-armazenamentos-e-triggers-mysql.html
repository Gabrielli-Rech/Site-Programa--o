Exercícios de Procedimentos, Armazenamentos e Triggers

Exercícios Fáceis

  • Crie um novo repositório Git e faça um commit inicial.
  • Crie um branch chamado `feature` e faça alterações simples em um arquivo.
  • Mescle o branch `feature` de volta para a branch `main` usando `git merge`.
  • Crie um branch chamado `bugfix` a partir da branch `main` e corrija um erro simples em um arquivo.
  • Use `git status` para verificar o estado do repositório após a mesclagem.

Exercícios Médios

  • Crie um conflito de mesclagem intencional entre os branches `main` e `feature` e resolva-o manualmente.
  • Use `git log` para verificar o histórico de commits após uma mesclagem.
  • Resolva um conflito de mesclagem usando o `git mergetool` e explore a ferramenta de resolução de conflitos.
  • Crie um branch `feature` e faça uma alteração em um arquivo. Em seguida, mude a mesma linha desse arquivo na branch `main` e tente mesclar as branches.
  • Pratique a utilização de `git diff` para verificar as diferenças entre os branches antes de mesclar.

Exercícios Difíceis

  • Simule um conflito entre as branches `main` e `feature` com alterações em múltiplos arquivos e resolva-os manualmente.
  • Explore o uso de `git rebase` para reescrever o histórico de commits e resolva conflitos durante o rebase.
  • Pratique o uso de `git cherry-pick` para aplicar um commit específico de um branch em outro e resolva conflitos se houver.
  • Experimente `git revert` em um commit que causou um conflito e analise como isso afeta o histórico do repositório.
  • Realize a fusão de três branches com alterações sobrepostas em um mesmo arquivo e resolva todos os conflitos resultantes.

Desafios

  • Crie um fluxo de trabalho que envolva criar um branch `feature`, fazer alterações em arquivos diferentes de outro colaborador, e mesclar ambos em `main` resolvendo os conflitos de forma colaborativa.
  • Utilize `git log --merge` para listar os commits que foram mesclados e analise os conflitos que foram resolvidos manualmente.
  • Use `git reflog` para verificar o histórico de mudanças no repositório e reverta um `git merge` se necessário.
  • Crie um cenário onde um `merge` automático falha e use a estratégia `ours` ou `theirs` para forçar a aceitação das alterações de um branch.
  • Desafie-se a criar um `patch` de uma alteração e aplicá-lo em outro branch, resolvendo possíveis conflitos.
