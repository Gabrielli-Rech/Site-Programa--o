Exercícios de Índices e Desempenho

  • Voltar
  • Início

Exercícios Fáceis

  • Crie um índice simples em um campo de uma coleção e verifique se ele foi criado com `db.collection.getIndexes()`.
  • Use o comando `db.collection.createIndex()` para criar um índice composto em dois campos de uma coleção.
  • Verifique a diferença de desempenho em consultas com e sem índice em uma coleção.
  • Use o comando `explain()` para analisar o plano de execução de uma consulta com e sem índice.
  • Crie um índice único para garantir a unicidade em um campo específico de uma coleção.
  • Remova um índice com o comando `db.collection.dropIndex()` e verifique se ele foi removido.
  • Verifique o uso de memória de índices com o comando `db.collection.stats()`.
  • Crie um índice em um campo que seja um array e use a consulta `$elemMatch` para buscar elementos específicos.
  • Teste o impacto de um índice em uma consulta de busca simples e uma consulta de projeção.
  • Verifique quais índices estão em uso em uma coleção com `db.collection.indexStats()`.

Exercícios Médios

  • Crie um índice composto em uma coleção e utilize a cláusula `db.collection.find().hint()` para forçar o uso desse índice.
  • Utilize o comando `db.collection.aggregate()` com uma etapa de `$lookup` e crie índices apropriados para otimizar o desempenho.
  • Analise o desempenho de consultas em uma coleção com diferentes tipos de índices: simples, compostos e de texto.
  • Use a estratégia de índices geoespaciais para otimizar consultas em campos de localização (`2dsphere`).
  • Crie um índice de texto e execute uma consulta de texto completo para avaliar seu desempenho.
  • Crie uma coleção de exemplo com dados e implemente um índice de TTL para expirar documentos automaticamente.
  • Utilize `db.collection.createIndex()` para criar um índice de fragmentação em um shard key em um cluster MongoDB.
  • Verifique o impacto da fragmentação em índices com `db.collection.stats()` e otimize-os se necessário.
  • Teste e compare a performance de consultas com índices criados em diferentes ordens de campos.
  • Implemente uma estratégia de índice para uma coleção com grande volume de dados e alta taxa de inserção.

Exercícios Difíceis

  • Desenvolva uma estratégia de indexação para uma coleção com múltiplos tipos de consultas e dados complexos.
  • Crie um índice para uma coleção que contenha dados de uma aplicação de análise em tempo real e verifique seu impacto.
  • Implemente e compare os índices de um campo com e sem índice de prefixo.
  • Testa o desempenho de consultas de agregação usando a etapa `$sort` e use índices adequados para otimizar.
  • Use `db.collection.aggregate()` para combinar e agrupar documentos com o suporte de índices e verifique o tempo de execução.
  • Crie e compare índices para dados com distribuição desigual e observe o efeito de `sparse` e `unique`.
  • Verifique o efeito de índices em operações de escrita e desenvolva estratégias para minimizar o impacto.
  • Implemente um sistema de monitoramento para identificar índices que não estão sendo usados com `db.collection.indexStats()`.
  • Desenvolva uma estratégia de cache para melhorar a performance de consultas em uma coleção de leitura intensiva.
  • Use o `db.collection.explain('executionStats')` para analisar o desempenho de consultas complexas e otimize com índices.

Desafios

  • Implemente uma estratégia de indexação híbrida para dados em um banco de dados de análise de logs e verifique o tempo de resposta.
  • Teste a combinação de índices de vários campos com diferentes estratégias de leitura e escrita.
  • Crie uma consulta de agregação complexa com etapas de `$match`, `$group` e `$sort` e otimize o desempenho com índices.
  • Implemente um sistema de gerenciamento de índices para automatizar a criação e remoção de índices com base em estatísticas de uso.
  • Desenvolva uma aplicação de teste para medir o impacto de diferentes configurações de índices em tempo real.
  • Crie uma coleção com documentos grandes e analise a performance de índices em consultas de busca por palavras-chave.
  • Desenvolva um processo para auditar índices em uso e criar relatórios sobre eficiência e necessidade de remoção.
  • Implemente uma política de expiração de índices e teste o impacto no desempenho após a remoção.
  • Utilize `db.collection.createIndex()` para criar um índice com `wildcard` em campos dinâmicos de documentos.
  • Testa a performance de consultas com um índice de `hashed` em comparação com um índice `B-tree` em um campo de chave primária.
