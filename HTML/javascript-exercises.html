Funções

Funções são blocos de código que realizam tarefas específicas e podem ser reutilizados em diferentes partes do programa. Pratique criando funções básicas e avançadas
Em JavaScript, funções são blocos de código que realizam tarefas específicas e podem ser reutilizados. Aqui está um resumo das principais
características e tipos de funções em JavaScript:
1. Declaração de Função
Funções podem ser definidas usando a palavra-chave function, seguidas de um nome e um corpo de código.
2. Expressões de Função
Funções podem ser atribuídas a variáveis, o que permite que sejam tratadas como valores.
3. Funções Anônimas e Arrow Functions
Funções anônimas são funções sem nome, geralmente usadas como valores de variáveis ou parâmetros. Arrow functions são uma forma concisa de escrever funções anônimas, usando a sintaxe de seta.
4. Funções com Parâmetros Padrão
Parâmetros podem ter valores padrão, que são usados caso nenhum valor seja passado para esses parâmetros.
5. Funções com Retorno
Funções podem retornar valores com a palavra-chave return, permitindo que o resultado da execução da função seja usado em outro lugar.
6. Funções Auto-Invocáveis (IIFE)
Funções que se executam automaticamente após serem definidas. São frequentemente usadas para criar escopos isolados.
7. Funções Assíncronas (Async/Await)
Funções que permitem o uso de async e await para lidar com operações assíncronas de forma mais legível e sequencial.
8. Funções de Callback
Funções que são passadas como argumentos para outras funções e executadas dentro delas.
9. Funções Recursivas
Funções que chamam a si mesmas para resolver um problema por meio de repetição.
10. Funções de Altas Ordens
Funções que podem receber outras funções como argumentos ou retorná-las como resultado.

Manipulação de Arrays e Objetos

Em JavaScript, um array é uma estrutura de dados que armazena uma lista de elementos em uma única variável. Ele é usado para armazenar coleções de dados, como números, strings e objetos.
Características de Arrays:
Indexação: Os elementos em um array são indexados a partir de 0.
Tipo dinâmico: Os arrays podem conter elementos de diferentes tipos de dados.
Métodos úteis: Métodos como push(), pop(), shift(), unshift(), map(), filter(), reduce() e outros permitem manipular os elementos de forma eficiente.
Array-List:
Array-List em JavaScript é frequentemente representado por arrays comuns, pois o JavaScript não possui uma implementação separada de ArrayList como em algumas outras linguagens de programação (por exemplo, Java). Arrays podem ser usados como listas dinâmicas que crescem ou diminuem de tamanho conforme a necessidade, o que os torna similares a uma ArrayList.
Resumo de Objetos em JavaScript
Objetos em JavaScript são coleções de pares de chave-valor e são a estrutura de dados mais usada para armazenar informações complexas. Eles são fundamentais para trabalhar com dados e para criar estruturas mais avançadas, como classes e instâncias.
Características de Objetos:
Chaves e valores: Cada propriedade de um objeto é uma chave única associada a um valor, que pode ser de qualquer tipo de dado (string, número, array, outro objeto, etc.).
Notação de objeto: Definidos usando chaves {} e separados por vírgulas.
Acesso às propriedades: As propriedades podem ser acessadas usando a notação de ponto (obj.propriedade) ou colchetes (obj['propriedade']).
Métodos: Objetos podem ter métodos, que são funções armazenadas como valores de uma chave.
Modificações: É possível adicionar, modificar e excluir propriedades de um objeto dinamicamente.
Exemplos de uso de objetos:
Representar dados de uma pessoa (nome, idade, endereço).
Criar instâncias de classes em JavaScript (funções construtoras).
Armazenar funções como propriedades e manipular dados complexos.
Comparação entre Arrays e Objetos
Arrays são mais adequados para listas ordenadas de elementos e para situações em que a ordem é importante. Objetos são ideais para representar dados com chaves únicas e valores associados, como entidades com propriedades distintas.

Funções Assíncronas

Funções assíncronas em JavaScript são usadas para lidar com operações que podem levar algum tempo para serem concluídas, como chamadas a APIs, leitura de arquivos, ou temporizadores. Elas permitem que o código continue executando outras tarefas enquanto aguardam a conclusão de operações assíncronas, melhorando a performance e a experiência do usuário.
Características das Funções Assíncronas:
Palavra-chave async: Uma função é declarada como assíncrona adicionando a palavra-chave async antes da declaração da função. Isso faz com que a função retorne automaticamente uma Promise.
Palavra-chave await: Dentro de uma função async, você pode usar await antes de uma expressão que retorna uma Promise para pausar a execução da função até que a Promise seja resolvida ou rejeitada. O await só pode ser usado dentro de funções async.
Retorno de funções async: Mesmo que você use return em uma função async, ela sempre retornará uma Promise. O valor retornado é automaticamente resolvido como o valor da Promise.
Benefícios das Funções Assíncronas:
Sintaxe mais legível: Com async/await, o código assíncrono parece e se comporta de forma mais parecida com o código síncrono, tornando-o mais fácil de ler e entender.
Tratamento de erros: A estrutura de try...catch pode ser usada dentro de funções async para capturar erros de maneira mais clara e organizada, em comparação com o uso de .catch() em Promises.
Fluxo de Execução:
Quando a função async é chamada, ela retorna uma Promise.
A execução da função é pausada na expressão await até que a Promise associada seja resolvida ou rejeitada.
Após a resolução da Promise, a execução da função continua normalmente.
Se a Promise for rejeitada, o fluxo é transferido para o bloco catch (se presente) para tratamento de erros.
Quando Usar Funções Assíncronas:
Quando você precisa fazer chamadas de rede (por exemplo, usando fetch()).
Ao lidar com operações de leitura e escrita de arquivos em ambientes como Node.js.
Em qualquer situação em que o código possa se beneficiar de uma execução não bloqueante.
Funções assíncronas são uma ferramenta poderosa para escrever código JavaScript que é mais eficiente e fácil de entender, especialmente em aplicações que dependem de operações de I/O, como APIs e banco de dados.
