Exercícios sobre Terminais do Visual Studio Code.

  • Voltar
  • Início
  • Abra o terminal integrado no VSCode usando o atalho `Ctrl + ``.
  • Crie um novo terminal com o comando `Terminal > Novo Terminal` ou use o atalho `Ctrl + Shift + '`.
  • Troque entre terminais abertos usando `Ctrl + Tab` ou clicando no terminal desejado na barra de terminais.
  • Use o comando `cd` para navegar para diferentes diretórios no terminal.
  • Liste os arquivos e pastas de um diretório usando `ls` (Linux/macOS) ou `dir` (Windows).
  • Execute um script de Python, Node.js ou outro script diretamente no terminal com o comando `python script.py` ou `node script.js`.
  • Saia do terminal ou encerre uma sessão de terminal usando o comando `exit`.
  • Use o comando `clear` ou `cls` para limpar a tela do terminal e manter a área de trabalho organizada.
  • Personalize o terminal do VSCode alterando a fonte, o tema e as cores em `Arquivo > Preferências > Configurações`.
  • Crie um arquivo e edite-o diretamente no terminal com o comando `nano arquivo.txt` (Linux/macOS) ou `notepad arquivo.txt` (Windows).
  • Redirecione a saída de um comando para um arquivo com `echo "Texto de exemplo" > arquivo.txt`.
  • Use o comando `cat arquivo.txt` para exibir o conteúdo de um arquivo no terminal.
  • Monitore o uso de recursos do sistema usando comandos como `top` (Linux/macOS) ou `tasklist` (Windows).
  • Utilize o comando `git status` para verificar o status de um repositório Git diretamente no terminal.
  • Execute comandos em segundo plano com `&` (exemplo: `npm start &`).
  • Verifique o caminho atual do diretório usando o comando `pwd` (Linux/macOS) ou `cd` (Windows).
  • Use o comando `history` para exibir o histórico de comandos executados no terminal.
  • Use o comando `grep` para buscar por palavras específicas em arquivos, como `grep "texto" arquivo.txt`.
  • Interrompa a execução de um processo em execução no terminal com `Ctrl + C`.
  • Use `npm install` para instalar pacotes do Node.js diretamente no terminal do VSCode.
  • Execute scripts de build ou testes automatizados diretamente do terminal do VSCode.
  • Configure um terminal integrado para abrir em uma pasta específica ao iniciar o VSCode, indo em `Configurações > Terminal > Abrir na pasta`.
  • Use a extensão `Remote - WSL` para trabalhar com terminais de subsistema Linux dentro do VSCode.
  • Experimente comandos avançados como `find` e `xargs` para buscar e manipular arquivos no terminal.
