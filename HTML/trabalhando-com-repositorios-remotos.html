Exercícios sobre Repositórios Remotos

  • Voltar
  • Início
  • Crie um repositório remoto no GitHub e clone-o para sua máquina local usando `git clone`.
  • Faça alterações em um arquivo local, faça um `commit` e use `git push origin main` para enviar as alterações para o repositório remoto.
  • Use `git pull origin main` para atualizar sua branch local com as últimas mudanças do repositório remoto.
  • Adicione um repositório remoto adicional com `git remote add` e envie suas mudanças com `git push` para esse novo repositório.
  • Verifique a URL do repositório remoto com `git remote -v` e altere-a com `git remote set-url` se necessário.
  • Use `git fetch` para baixar atualizações do repositório remoto sem fazer merge automático.
  • Crie uma branch `feature/remote-work` e envie-a para o repositório remoto com `git push origin feature/remote-work`.
  • Faça `git pull --rebase` para atualizar sua branch local com alterações do repositório remoto e evitar um merge automático.
  • Use `git cherry-pick` para aplicar um commit específico do repositório remoto na sua branch local.
  • Resolva conflitos ao fazer `git pull` com um repositório remoto e faça um `commit` para concluir a mesclagem.
  • Verifique se há branches remotas com `git branch -r` e faça um `checkout` para uma branch remota com `git checkout -b`.
  • Sincronize seu repositório local com o remoto usando `git pull` e, em seguida, `git push` para garantir que as mudanças estejam atualizadas.
  • Utilize `git push --force` para forçar a atualização de uma branch remota, mas tome cuidado com esse comando para não sobrescrever alterações de outros colaboradores.
  • Use `git remote show origin` para ver detalhes sobre o repositório remoto, como branches rastreadas e URLs.
  • Configure um repositório remoto como `upstream` em um fork com `git remote add upstream `, e use `git fetch upstream` para sincronizar.
  • Faça um `git push --set-upstream origin feature/new-feature` para associar uma branch local a uma branch remota de mesmo nome.
  • Verifique se existem mudanças locais não enviadas para o repositório remoto com `git status` e use `git diff` para comparar.
  • Use `git log origin/main..HEAD` para visualizar os commits que você ainda não enviou para o repositório remoto.
  • Faça um `git rebase` em uma branch local para aplicar suas alterações sobre a branch remota principal.
  • Use `git tag` para marcar um ponto específico na história de um repositório e faça `git push origin ` para enviar a tag para o repositório remoto.
  • Configure um repositório remoto para ser somente leitura com `git remote set-url --push origin `, se necessário.
  • Use `git pull --no-ff` para fazer um merge com um commit de merge explícito ao integrar uma branch remota na sua branch local.
  • Verifique e compare a diferença entre sua branch local e a branch remota com `git diff origin/main`.
