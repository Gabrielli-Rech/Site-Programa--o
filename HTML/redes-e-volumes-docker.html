Exercícios de Redes e Volumes Docker

  • Crie uma rede Docker personalizada usando o comando `docker network create` e conecte dois containers a ela.
  • Verifique as redes existentes com o comando `docker network ls` e detalhes da rede com `docker network inspect`.
  • Crie e execute um container em uma rede específica usando a opção `--network`.
  • Teste a comunicação entre dois containers na mesma rede usando o comando `ping`.
  • Crie uma rede do tipo `bridge`, `host` e `overlay` e teste as diferenças entre elas.
  • Configure um container com IP fixo em uma rede Docker personalizada.
  • Conecte um container a uma rede existente e depois desconecte-o usando `docker network disconnect`.
  • Crie um container com múltiplas interfaces de rede, conectando-o a redes diferentes.
  • Configure o modo de comunicação entre containers usando a opção `--link` (deprecated) e teste a conectividade.
  • Teste a conexão entre containers em redes diferentes e identifique o comportamento de isolamento.
  • Crie volumes Docker usando o comando `docker volume create` e liste todos os volumes com `docker volume ls`.
  • Monte um volume em um container e verifique se as alterações persistem após a reinicialização do container.
  • Configure volumes nomeados e anônimos e compare seus usos e benefícios.
  • Use um volume Docker para compartilhar dados entre dois containers em execução.
  • Mapeie um volume local para um diretório do container e teste a leitura e escrita de arquivos.
  • Remova um volume Docker que não está mais em uso com o comando `docker volume rm`.
  • Monte volumes de apenas leitura em um container para garantir que os dados não possam ser modificados.
  • Crie um volume que use um driver específico, como `local` ou `nfs`, e configure-o com opções apropriadas.
  • Use o comando `docker-compose` para definir volumes e redes em um arquivo `docker-compose.yml` e execute o projeto.
  • Monitore o uso de volumes e redes com o comando `docker system df` para verificar a utilização de armazenamento.
  • Crie um container com um volume que armazene logs e configure o container para registrar informações nesse volume.
  • Verifique e compare as permissões de acesso aos volumes Docker usando os comandos de leitura e escrita.
