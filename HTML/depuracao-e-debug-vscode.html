Exercícios Deupuração e Debug.

Exercícios Fáceis

  • 1. Configurando o Debugger: Instale a extensão do Visual Studio Code para depuração de JavaScript e configure um arquivo launch.json.
  • 2. Breakpoints Simples: Adicione um breakpoint em uma linha de código e inicie o modo de depuração.
  • 3. Usando o Console de Debug: Execute um código com um breakpoint e utilize o console de depuração para inspecionar variáveis.
  • 4. Verificando Variáveis: Use a aba de variáveis do VSCode para visualizar o valor de uma variável durante a execução.
  • 5. Passos de Execução: Teste os botões de "Step Over" e "Step Into" durante a depuração.
  • 6. Depurando um Loop: Coloque um breakpoint em um loop e use a execução passo a passo para entender seu fluxo.
  • 7. Inspecionando Objetos: Verifique a estrutura de um objeto durante a depuração.
  • 8. Configurando Watch: Adicione uma expressão à aba "Watch" para monitorar uma variável específica.
  • 9. Erro de Sintaxe: Corrija um erro de sintaxe simples e observe como o depurador identifica o problema.
  • 10. Mensagens de Log: Adicione console.log em partes específicas do código e observe os resultados na aba de depuração.

Exercícios Médios

  • 11. Debug de Aplicação em Node.js: Crie um script simples em Node.js e depure-o para encontrar o motivo de um erro.
  • 12. Ponto de Interrupção Condicional: Configure um ponto de interrupção que só seja acionado quando uma condição específica for atendida.
  • 13. Debugando com Módulos Externos: Utilize a depuração para rastrear como um módulo externo está sendo utilizado.
  • 14. Analisando Stacks de Erro: Use a aba de chamadas de pilha para identificar o caminho que levou a um erro.
  • 15. Depuração de Código Assíncrono: Adicione breakpoints em um código com promessas e observe como ele é executado.
  • 16. Depuração de Funções Anônimas: Coloque um breakpoint em uma função anônima e entenda como depurá-la.
  • 17. Execução de Testes Unitários: Configure o VSCode para depurar testes unitários e encontre falhas em suas funções.
  • 18. Visualização de Eventos de Depuração: Utilize a extensão "Debugger for Chrome" para depurar uma aplicação front-end.
  • 19. Explorando a Interface do Debugger: Familiarize-se com as diferentes seções da interface de depuração (variáveis, watch, call stack).
  • 20. Analisando Performance: Use as ferramentas de perfil de execução do VSCode para identificar pontos de lentidão no código.

Exercícios Difíceis

  • 21. Debugando Aplicações com Conexões de Banco de Dados: Depure uma aplicação que se conecta a um banco de dados para rastrear consultas lentas.
  • 22. Depuração de Erros de Race Condition: Identifique e resolva um problema de race condition em uma aplicação multi-thread.
  • 23. Depuração de Código Minificado: Trabalhe com um arquivo de código minificado e use a fonte map para depurá-lo.
  • 24. Integrando com Contêineres Docker: Configure a depuração de uma aplicação rodando em um contêiner Docker.
  • 25. Debugando em Tempo Real com WebSockets: Depure uma aplicação que utiliza WebSockets para comunicação em tempo real.
  • 26. Configuração Avançada de launch.json: Crie um arquivo de configuração launch.json complexo com variáveis de ambiente.
  • 27. Debugando com Extensões: Use uma extensão para depurar aplicações específicas, como React ou Vue.js.
  • 28. Debugando em Aplicações Distribuídas: Trabalhe na depuração de uma aplicação que utiliza microserviços.
  • 29. Rastreamento de Memory Leaks: Identifique e corrija vazamentos de memória em uma aplicação.
  • 30. Analisando Performance de Código: Utilize ferramentas de profiling no VSCode para encontrar e corrigir gargalos de performance.

Desafios

  • 31. Integrando Depuração com CI/CD: Configure um pipeline de CI/CD para iniciar a depuração em casos de falhas de testes.
  • 32. Depuração em um Ambiente de Produção: Adicione uma configuração de depuração remota para uma aplicação rodando em produção.
  • 33. Debugando com Testes de Integração: Depure testes de integração em uma aplicação complexa para rastrear erros interdependentes.
  • 34. Desafio de Performance de Código: Utilize ferramentas de profiling no VSCode para encontrar e corrigir gargalos de performance.
  • 35. Debugando Aplicação com WebAssembly: Configure a depuração de uma aplicação que usa WebAssembly.
  • 36. Trabalhando com Depuração de Testes E2E: Depure uma suíte de testes end-to-end e encontre falhas em interações complexas.
  • 37. Configuração de Hot Reload: Configure um ambiente de desenvolvimento para depuração com hot reload em um projeto front-end.
  • 38. Analisando Logs de Depuração: Use as ferramentas de depuração para interpretar e analisar logs complexos.
  • 39. Depuração de Problemas de Conectividade de Rede: Depure uma aplicação que apresenta problemas de conectividade com APIs externas.
  • 40. Debugando Threads em Multithreading: Configure a depuração de múltiplas threads para encontrar e resolver problemas de concorrência.
