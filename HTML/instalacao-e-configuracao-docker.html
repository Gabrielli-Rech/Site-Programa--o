O que é o Docker?

O Docker é uma plataforma que permite desenvolver, enviar e executar aplicações em containers. Containers são leves e isolados, permitindo que você execute aplicações em qualquer ambiente de forma consistente.

Instalando o Docker

A instalação do Docker pode variar de acordo com o sistema operacional. Veja como instalar o Docker nos principais sistemas:

Instalação no Windows

Siga estes passos para instalar o Docker no Windows:

  1. Certifique-se de que o Windows 10 ou versão superior está atualizado.
  2. Baixe o Docker Desktop para Windows em docker.com.
  3. Execute o instalador e siga as instruções na tela.
  4. Após a instalação, inicie o Docker Desktop e aguarde a inicialização completa.

Instalação no macOS

Para instalar o Docker no macOS, siga os passos abaixo:

  1. Certifique-se de que o macOS 10.14 ou superior está atualizado.
  2. Baixe o Docker Desktop para macOS em docker.com.
  3. Execute o instalador e siga as instruções na tela.
  4. Após a instalação, inicie o Docker Desktop e aguarde a inicialização completa.

Instalação no Linux

Para instalar o Docker no Linux, siga os passos para sua distribuição:

  • Ubuntu: sudo apt-get update e sudo apt-get install docker.io
  • CentOS: sudo yum install docker

Inicie o serviço do Docker com: sudo systemctl start docker

Configurando o Docker

Após a instalação, é importante configurar o Docker para otimizar seu uso:

  1. Verifique a instalação com o comando docker --version para garantir que o Docker foi instalado corretamente.
  2. Execute o comando docker run hello-world para verificar se o Docker está funcionando corretamente.
  3. Para facilitar o uso, você pode adicionar seu usuário ao grupo do Docker com:
sudo usermod -aG docker $USER

Após executar esse comando, reinicie o sistema ou faça logout e login novamente para que as alterações tenham efeito.

Executando Containers Docker

Para executar um container do Docker, use o comando docker run. Por exemplo:

docker run -d -p 80:80 nginx

Este comando executa um container do Nginx em segundo plano e mapeia a porta 80 do container para a porta 80 do host.
