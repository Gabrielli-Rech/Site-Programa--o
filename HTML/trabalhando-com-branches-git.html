Exercícios de Criação e Manipulação de Branches.

  • Voltar
  • Início
  • Crie um novo repositório Git local e uma branch chamada `feature/login`.
  • Crie uma branch chamada `feature/registration` a partir da branch `main` e faça um commit com uma mensagem descritiva.
  • Troque para a branch `feature/login` e faça alterações em um arquivo. Depois, faça um commit dessas alterações.
  • Faça um `merge` da branch `feature/login` na branch `main` e resolva possíveis conflitos.
  • Crie uma branch chamada `bugfix/fix-header` e corrija um erro no arquivo de cabeçalho do projeto. Depois, faça um `merge` dessa branch com a `main`.
  • Crie uma branch `feature/user-profile` e, após terminar o desenvolvimento, faça o `rebase` dela com a branch `main` para atualizar com as últimas mudanças.
  • Crie uma branch `feature/dashboard` e faça um `cherry-pick` de um commit específico de outra branch para essa nova branch.
  • Faça um `checkout -b` para criar e mudar para uma nova branch chamada `feature/settings` em um único comando.
  • Crie uma branch de teste `test/feature-test` e faça mudanças que não sejam necessárias para o código principal. Em seguida, exclua a branch após o teste.
  • Crie uma branch de desenvolvimento chamada `dev` e faça alterações que contenham código experimental. Depois, faça um `merge` controlado para a `main`.
  • Crie uma branch chamada `hotfix/critical-bug` a partir de `main` e faça o `merge` dessa branch de volta à `main` após a correção.
  • Use o comando `git log --oneline --graph` para visualizar as branches e seus históricos de commit.
  • Crie uma branch `feature/new-feature` a partir da `main`, faça alterações e depois use `git diff` para comparar com a branch `main` antes do merge.
  • Use `git rebase` interativamente para reescrever o histórico de commits em uma branch `feature/cleanup`.
  • Verifique se há branches remotas e faça o `fetch` para atualizar as referências de branches remotas.
  • Crie e faça checkout para uma branch `feature/bug-report`, faça um commit e depois use `git push origin feature/bug-report` para enviar para o repositório remoto.
  • Use `git branch -d` para excluir uma branch local que já foi mesclada com `main`.
  • Crie uma branch `feature/collaboration` e trabalhe em equipe para fazer `merge` e resolver conflitos.
  • Crie uma branch `feature/test-feature` e use `git stash` para salvar temporariamente as mudanças antes de mudar para outra branch.
  • Use `git checkout main` e `git merge feature/update-docs` para fazer o `merge` da branch `feature/update-docs` na `main`.
  • Crie uma branch `feature/refactor` e faça alterações em uma função específica. Depois, use `git status` para verificar o status da branch.
