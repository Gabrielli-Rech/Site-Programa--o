Exercícios de Publicação e Registro de Imagens

  • Voltar
  • Início
  • Crie um arquivo Dockerfile simples para construir uma imagem de uma aplicação "Hello World".
  • Construa a imagem Docker usando o comando `docker build` e tagueie-a com um nome específico.
  • Verifique as imagens criadas localmente usando o comando `docker images`.
  • Teste a imagem localmente com o comando `docker run` e verifique se a aplicação está rodando.
  • Crie uma conta em um registro Docker como Docker Hub, GitHub Container Registry ou Amazon ECR.
  • Faça login no registro Docker usando `docker login` e insira as credenciais da conta.
  • Tagueie a imagem local com o formato correto para o registro Docker, por exemplo: `docker tag /my-image:latest`.
  • Publique a imagem no registro Docker usando o comando `docker push`.
  • Verifique se a imagem foi publicada com sucesso acessando o painel do registro Docker.
  • Faça o download da imagem de um registro Docker em outra máquina usando `docker pull`.
  • Crie uma imagem Docker com variáveis de ambiente e faça a publicação no registro.
  • Teste a execução de uma imagem publicada em outro ambiente (máquina local diferente ou servidor remoto).
  • Configure o `Dockerfile` para incluir múltiplas etapas e minimize o tamanho da imagem.
  • Adicione um arquivo `.dockerignore` ao projeto para otimizar a construção da imagem Docker.
  • Use o comando `docker history ` para ver a camada de construção da imagem.
  • Crie um script de automação para a construção, tag e publicação de imagens Docker usando `bash` ou `PowerShell`.
  • Publique uma imagem Docker privada em um registro privado (exemplo: Amazon ECR, Azure Container Registry, etc.).
  • Verifique a política de retenção de imagens no registro Docker e configure-a conforme necessário.
  • Realize a autenticação automática com um token de acesso em vez de uma senha com o comando `docker login`.
  • Automatize a publicação de uma imagem Docker em um pipeline de CI/CD (GitHub Actions, GitLab CI/CD, Jenkins, etc.).
  • Crie uma imagem multi-plataforma (Linux e Windows) e publique-a em um registro Docker.

Bom Estudo.
