Exercícios Fáceis

  1. Inicializar um repositório: Crie um novo repositório Git local usando git init.
  2. Verificar o status: Use o comando git status para ver o estado dos arquivos no repositório.
  3. Adicionar arquivos: Adicione um arquivo chamado index.html ao repositório com git add.
  4. Confirmar alterações: Faça um commit com a mensagem "Adicionando index.html" usando git commit.
  5. Ver histórico de commits: Visualize o histórico de commits com git log.
  6. Ver diferenças: Verifique as diferenças não confirmadas com git diff.
  7. Clonar repositório: Clone um repositório remoto com git clone.
  8. Criar branch: Crie uma nova branch chamada feature com git branch.
  9. Mudar de branch: Altere para a branch feature com git checkout.
  10. Ver branches: Liste todas as branches no repositório com git branch.

Exercícios Médios

  1. Comitar alterações específicas: Faça um commit de arquivos específicos com git add e git commit.
  2. Merge de branches: Realize o merge da branch feature na branch main com git merge.
  3. Excluir branch: Apague uma branch local com git branch -d.
  4. Rebase de branch: Use git rebase para aplicar alterações de uma branch em outra.
  5. Resolução de conflitos: Pratique a resolução de conflitos durante o merge.
  6. Desfazer alterações locais: Use git checkout -- <arquivo> para desfazer alterações em um arquivo específico.
  7. Remover arquivos do repositório: Use git rm <arquivo> para remover arquivos do repositório.
  8. Atualizar o repositório remoto: Use git push para enviar suas alterações para o repositório remoto.
  9. Obter mudanças do repositório remoto: Use git pull para buscar e integrar alterações do repositório remoto.
  10. Ver log com gráficos: Use git log --graph para visualizar o histórico de commits de forma gráfica.

Exercícios Difíceis

  1. Reverter um commit: Use git revert para criar um novo commit que desfaz um commit anterior.
  2. Resetar um commit: Use git reset para reverter para um estado anterior e ajustar o histórico de commits.
  3. Stash de alterações: Guarde alterações temporárias com git stash e recupere com git stash pop.
  4. Criar tags: Marque um commit específico com uma tag usando git tag.
  5. Buscar tags remotas: Use git fetch --tags para buscar tags de um repositório remoto.
  6. Rebase interativo: Use git rebase -i para reescrever o histórico de commits.
  7. Verificar o histórico de um arquivo: Use git log -- <arquivo> para ver o histórico de alterações de um arquivo específico.
  8. Comparar branches: Use git diff <branch1> <branch2> para ver as diferenças entre duas branches.
  9. Configurar repositório remoto: Adicione um repositório remoto com git remote add.
  10. Ver alterações no repositório: Use git show <commit> para visualizar detalhes de um commit específico.

Desafios

  1. Restaurar arquivo deletado: Restaure um arquivo deletado com git checkout.
  2. Apagar histórico de commits: Use git reset --hard para apagar alterações e commits locais.
  3. Aplicar patches: Use git apply para aplicar um patch de outro repositório.
  4. Ver o impacto de um commit: Use git diff <commit> para ver as alterações feitas em um commit específico.
  5. Navegar por commits antigos: Use git checkout <commit> para navegar por um commit antigo.
  6. Unir commits: Use git rebase -i para combinar múltiplos commits em um só.
  7. Filtrar commits por autor: Use git log --author="nome" para ver commits de um autor específico.
  8. Desfazer commit local: Use git reset --soft HEAD~1 para desfazer o último commit, mantendo as alterações no estágio.
  9. Ver mudanças em um arquivo: Use git diff <commit> <arquivo> para comparar um arquivo em um commit específico.
  10. Configurar alias: Crie um alias para um comando Git comum com git config --global alias.<alias>.
