O que é um Dockerfile?

Um Dockerfile é um arquivo de texto que contém todas as instruções necessárias para criar uma imagem Docker. Ele especifica o sistema base, as dependências, as configurações e os comandos que devem ser executados na imagem.

Estrutura de um Dockerfile

Um Dockerfile é composto por uma série de instruções que definem o processo de construção da imagem. As instruções mais comuns incluem:

  • FROM: Define a imagem base para a construção. Exemplo: FROM ubuntu:20.04.
  • RUN: Executa comandos dentro da imagem. Exemplo: RUN apt-get update && apt-get install -y nodejs.
  • WORKDIR: Define o diretório de trabalho dentro da imagem. Exemplo: WORKDIR /app.
  • COPY: Copia arquivos da máquina host para a imagem. Exemplo: COPY . ..
  • CMD: Define o comando a ser executado quando o container for iniciado. Exemplo: CMD ["node", "app.js"].

Exemplo de um Dockerfile básico:

FROM node:14
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 8080
CMD ["node", "app.js"]

Imagens Docker

As imagens Docker são arquivos binários que contêm tudo o que é necessário para executar um aplicativo, incluindo código, bibliotecas, dependências, variáveis de ambiente e configurações de sistema.

As imagens são criadas a partir de um Dockerfile e podem ser armazenadas em registries, como o Docker Hub, para serem compartilhadas e reutilizadas.

Comandos básicos para trabalhar com imagens Docker:

  • docker build: Cria uma imagem Docker a partir de um Dockerfile. Exemplo: docker build -t minha-imagem ..
  • docker images: Lista as imagens Docker disponíveis no sistema. Exemplo: docker images.
  • docker rmi: Remove uma imagem Docker. Exemplo: docker rmi minha-imagem.
