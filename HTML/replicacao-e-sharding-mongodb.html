Exercícios de Replicação e Sharding

  • Voltar
  • Início
  • Crie uma configuração de replicação simples com um único nó primário e um nó secundário no MongoDB.
  • Configure um conjunto de réplicas com três nós (primário e dois secundários) e verifique o status com `rs.status()`.
  • Configure um nó secundário como um nó de leitura para balancear a carga de leitura no cluster.
  • Simule uma falha do nó primário e observe o processo de eleição do novo nó primário.
  • Configure a replicação de dados com um nó arbiter para manter o número ímpar de votos em um conjunto de réplicas.
  • Adicione um novo nó secundário a um conjunto de réplicas existente e verifique sua sincronização com `rs.printReplicationInfo()`.
  • Teste a consistência de leitura e escrita em um conjunto de réplicas configurando diferentes níveis de leitura e escrita.
  • Configure um cluster de sharding simples com um shard e um config server no MongoDB.
  • Habilite o sharding em uma base de dados e selecione a chave de shard apropriada para uma coleção.
  • Adicione um novo shard a um cluster existente e verifique a redistribuição de dados com `sh.status()`.
  • Teste a fragmentação de dados inserindo documentos que sejam particionados em shards diferentes.
  • Configure a replicação em um cluster sharded com shards replicados para alta disponibilidade.
  • Verifique o status do cluster de sharding com o comando `sh.status()` e analise as estatísticas de distribuição de shards.
  • Configure o balancer para gerenciar a distribuição de dados entre shards e analise seu impacto na performance.
  • Examine as opções de política de chunking em sharding e ajuste a configuração para otimizar a distribuição de dados.
  • Configure um shard com particionamento manual e analise a carga e desempenho com o comando `db.collection.getShardDistribution()`.
  • Desative temporariamente o balancer para evitar que ele migre chunks durante operações críticas.
  • Testa a consulta de dados distribuídos em um cluster sharded e analise a performance de uma consulta com `explain()`.
  • Configure um cluster de replicação com shards em regiões geográficas diferentes para alta disponibilidade global.
  • Implemente uma política de backups para dados em um cluster sharded e verifique a consistência dos backups.
  • Monitore o desempenho do cluster sharded usando ferramentas como `mongostat` e `mongotop`.
  • Crie e aplique índices em uma coleção sharded e observe o impacto na performance de leitura e escrita.
  • Realize operações de manutenção em um cluster sharded, como compactação de dados e verificação de integridade.
