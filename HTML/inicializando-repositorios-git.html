O que é um Repositório Git?

Um repositório Git é um local onde os arquivos do projeto são armazenados e gerenciados usando o sistema de controle de versão Git. Ele permite que você acompanhe as mudanças, colabore com outros desenvolvedores e gerencie diferentes versões do código de forma eficiente.

Como Inicializar um Repositório Git

Para criar um repositório Git em um diretório de projeto existente, você precisa usar o comando git init.

Passos para inicializar um repositório:

  1. Navegue até o diretório do projeto usando o terminal ou prompt de comando.
  2. Execute o comando git init.
  3. Isso criará um subdiretório chamado .git, que contém todos os arquivos necessários para o repositório.

Exemplo de inicialização de repositório:

cd meu-projeto
git init

Adicionando e Comitando Arquivos

Depois de inicializar o repositório, você pode adicionar arquivos e fazer o primeiro commit.

Para adicionar arquivos, use:

git add .

Para fazer o commit, use:

git commit -m "Meu primeiro commit"

Explicação:

  • git add .: Adiciona todos os arquivos do diretório atual para serem rastreados pelo Git.
  • git commit -m "Mensagem": Cria um novo commit com os arquivos adicionados e uma mensagem descritiva.

Conectando ao Repositório Remoto

Para conectar o repositório local a um repositório remoto, use o comando git remote add.

Exemplo de comando para conectar a um repositório remoto:

git remote add origin https://github.com/usuario/meu-repositorio.git

Para enviar as alterações para o repositório remoto, use:

git push -u origin master

Explicação:

  • git remote add origin: Define a URL do repositório remoto chamado "origin".
  • git push -u origin master: Envia as alterações da branch "master" para o repositório remoto "origin".
