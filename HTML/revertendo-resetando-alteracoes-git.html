Exercícios de Reversão e Reset de Alterações

  • Voltar
  • Início
  • Crie um repositório Git e adicione um arquivo de teste. Faça uma alteração e use `git status` para verificar as mudanças.
  • Use `git checkout -- ` para reverter uma alteração em um arquivo específico antes de comitar.
  • Faça um commit e depois use `git reset --hard HEAD~1` para desfazer o último commit. Verifique o histórico com `git log`.
  • Use `git reset --soft HEAD~1` para desfazer o último commit, mas manter as alterações na área de stage.
  • Utilize `git revert ` para criar um novo commit que desfaz as alterações de um commit específico. Verifique o histórico com `git log`.
  • Faça uma alteração em um arquivo e use `git stash` para salvar as alterações temporariamente. Use `git stash apply` para reaplicar as mudanças.
  • Faça um commit e, em seguida, use `git reset --mixed HEAD~1` para remover o último commit, mantendo as alterações no diretório de trabalho.
  • Teste a reversão de um arquivo específico com `git checkout -- ` para restaurar a versão do arquivo de um commit anterior.
  • Use `git reflog` para encontrar o hash de um commit perdido e use `git reset --hard ` para restaurar esse commit.
  • Desfaça um commit e suas alterações com `git reset --hard ` e certifique-se de que o commit e as alterações desapareceram do histórico.
  • Faça uma alteração, comite e, em seguida, use `git cherry-pick ` para aplicar as alterações de um commit específico em sua branch atual.
  • Utilize `git diff` para visualizar as alterações que não foram comitadas antes de reverter ou resetar.
  • Crie uma branch de teste e use `git reset --hard` para entender o impacto de um reset em uma branch específica.
  • Reverta um merge com `git reflog` e `git reset --hard ` para desfazer um merge indesejado.
  • Use `git log --oneline` para visualizar a lista de commits e identificar qual commit reverter.
  • Utilize `git rebase` para fazer uma reescrita do histórico de commits e depois use `git reset` para ajustes finos.
  • Teste como `git reset --hard` interage com arquivos não rastreados no repositório.
  • Desfaça alterações em um arquivo específico, mas mantenha o restante com `git checkout -- `.
  • Pratique a reversão de múltiplos commits usando `git revert ..`.
  • Use `git clean -f` para remover arquivos não rastreados no diretório de trabalho e entenda como ele se comporta com `git reset`.
  • Compreenda a diferença entre `git reset --hard`, `git reset --soft`, e `git checkout -- ` para reverter alterações.
  • Teste a restauração de um commit específico com `git checkout` e depois como ele afeta sua branch com `git reset`.
