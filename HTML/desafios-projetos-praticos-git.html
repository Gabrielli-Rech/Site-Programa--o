Desafios Fáceis

  • 1. Criar um repositório local e fazer o primeiro commit de um arquivo README.md.
  • 2. Clonar um repositório remoto do GitHub e explorar os arquivos locais.
  • 3. Fazer alterações em um arquivo local, adicionar ao stage e realizar um commit.
  • 4. Criar uma branch nova e mudar para essa branch.
  • 5. Fazer um merge de uma branch em outra localmente.
  • 6. Usar o comando git status para verificar o status do repositório.
  • 7. Criar tags em um commit específico e visualizar as tags com git tag.
  • 8. Reverter um arquivo para a versão do último commit usando git checkout.
  • 9. Configurar o nome de usuário e e-mail do Git globalmente com git config.
  • 10. Usar o comando git log para visualizar o histórico de commits.

Desafios Médios

  • 11. Criar um repositório remoto no GitHub e conectá-lo a um repositório local com git remote add.
  • 12. Usar git pull para atualizar um repositório local com as últimas alterações do repositório remoto.
  • 13. Resolver conflitos de merge em um repositório e realizar um commit com as alterações resolvidas.
  • 14. Usar git stash para guardar mudanças temporárias e recuperá-las posteriormente.
  • 15. Criar um repositório com múltiplas branches e fazer um merge de uma branch para a branch principal.
  • 16. Rebase de uma branch em relação a outra e resolver possíveis conflitos.
  • 17. Criar um arquivo .gitignore para excluir arquivos específicos do repositório.
  • 18. Configurar o SSH para autenticar com um repositório remoto no GitHub.
  • 19. Usar git diff para comparar as mudanças entre commits.
  • 20. Realizar um cherry-pick de um commit específico de outra branch.

Desafios Difíceis

  • 21. Configurar um repositório com hooks personalizados para verificar formatação de código antes do commit.
  • 22. Usar git rebase -i para reorganizar, modificar e remover commits em uma branch.
  • 23. Criar e gerenciar submódulos em um repositório Git para compartilhar código entre projetos.
  • 24. Reverter um commit específico em uma branch e manter o histórico de commits intacto com git revert.
  • 25. Realizar um fork de um repositório, fazer alterações e criar um pull request para o repositório original.
  • 26. Criar e configurar um repositório Git em um servidor local com acesso via SSH.
  • 27. Usar git bisect para encontrar o commit que introduziu um bug em uma branch.
  • 28. Usar git blame para identificar quem fez a última modificação em uma linha específica de um arquivo.
  • 29. Configurar um repositório para usar uma política de commit padrão com commit-msg hook.
  • 30. Realizar um git reset para desfazer um commit e mudar as alterações para o stage.

Desafios Avançados

  • 31. Configurar um pipeline de CI/CD com GitLab CI/CD para automatizar testes e builds de código.
  • 32. Usar git filter-branch para modificar o histórico de commits de um repositório.
  • 33. Implementar um sistema de controle de versão para um projeto em Docker usando o Git.
  • 34. Integrar Git com uma ferramenta de visualização de gráficos de commits, como Gitk ou Git Graph.
  • 35. Realizar uma migração de um repositório Git local para um repositório remoto em um servidor privado.
  • 36. Usar git reflog para encontrar e restaurar referências perdidas em um repositório.
  • 37. Configurar e usar Git Hooks para automatizar a execução de scripts após eventos específicos, como post-commit.
  • 38. Usar git archive para criar um arquivo compactado com uma versão específica do repositório.
  • 39. Criar um repositório com Git LFS para gerenciar arquivos grandes de forma eficiente.
  • 40. Criar um dashboard de visualização de métricas de um repositório usando uma API de GitHub e ferramentas de análise de dados.
