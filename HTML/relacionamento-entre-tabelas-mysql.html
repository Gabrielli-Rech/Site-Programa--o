O que é Relacionamento entre Tabelas?

Em um banco de dados relacional como o MySQL, as tabelas podem se relacionar entre si para modelar melhor os dados e garantir a integridade referencial. Esses relacionamentos permitem que você combine informações de diferentes tabelas de maneira lógica.

Tipos de Relacionamento entre Tabelas

Os principais tipos de relacionamento entre tabelas são:

  • Relacionamento um-para-um (1:1): Um registro em uma tabela corresponde a um único registro em outra tabela. Exemplo: cada pessoa pode ter um único documento de identidade.
  • Relacionamento um-para-muitos (1:N): Um registro em uma tabela pode ter muitos registros correspondentes em outra tabela. Exemplo: um cliente pode fazer muitos pedidos.
  • Relacionamento muitos-para-muitos (N:N): Muitos registros em uma tabela podem se relacionar com muitos registros em outra tabela. Exemplo: um aluno pode se matricular em várias disciplinas e uma disciplina pode ter vários alunos. Para isso, é necessário criar uma tabela intermediária.

Exemplo de Relacionamento em MySQL

Veja como criar um relacionamento entre tabelas com chave primária e chave estrangeira:

CREATE TABLE clientes (
    cliente_id INT PRIMARY KEY,
    nome VARCHAR(100) NOT NULL
);

CREATE TABLE pedidos (
    pedido_id INT PRIMARY KEY,
    cliente_id INT,
    data_pedido DATE,
    FOREIGN KEY (cliente_id) REFERENCES clientes(cliente_id)
);

Neste exemplo, a tabela pedidos tem uma chave estrangeira cliente_id que faz referência à chave primária da tabela clientes. Isso cria um relacionamento um-para-muitos entre clientes e pedidos.

Exemplo de Relacionamento Muitos-para-Muitos

Para criar um relacionamento muitos-para-muitos, você precisa de uma tabela intermediária:

CREATE TABLE alunos (
    aluno_id INT PRIMARY KEY,
    nome VARCHAR(100) NOT NULL
);

CREATE TABLE disciplinas (
    disciplina_id INT PRIMARY KEY,
    nome VARCHAR(100) NOT NULL
);

CREATE TABLE matriculas (
    aluno_id INT,
    disciplina_id INT,
    PRIMARY KEY (aluno_id, disciplina_id),
    FOREIGN KEY (aluno_id) REFERENCES alunos(aluno_id),
    FOREIGN KEY (disciplina_id) REFERENCES disciplinas(disciplina_id)
);

Neste exemplo, a tabela matriculas é usada para criar a associação entre os alunos e as disciplinas, permitindo que um aluno se matricule em várias disciplinas e uma disciplina tenha vários alunos.
