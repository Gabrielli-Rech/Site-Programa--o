Exercícios de Inicialização de Repositórios

  • Crie um novo repositório Git local em um diretório chamado "meu-projeto".
  • Inicialize um repositório Git em um diretório existente que já contém arquivos.
  • Verifique o status de um repositório após inicializá-lo.
  • Crie um arquivo chamado "README.md" e adicione um título ao projeto usando a ferramenta de edição de texto de sua escolha. Em seguida, adicione-o ao repositório com o comando `git add`.
  • Faça o primeiro commit no repositório com uma mensagem descritiva como "Inicializando o projeto".
  • Verifique o histórico de commits usando `git log`.
  • Crie um novo branch chamado "desenvolvimento" e mude para ele.
  • Crie um arquivo chamado "index.html" e faça o commit desse arquivo no branch "desenvolvimento".
  • Retorne para o branch principal e faça um merge do branch "desenvolvimento".
  • Crie um arquivo ".gitignore" para ignorar arquivos temporários e commitá-lo no repositório.
  • Configure seu nome de usuário e email no Git e verifique se está configurado corretamente com `git config`.
  • Verifique a diferença entre os arquivos modificados e o último commit usando `git diff`.
  • Crie um repositório remoto no GitHub, GitLab ou outro serviço, e conecte-o ao repositório local usando `git remote add`.
  • Faça um "push" do seu repositório local para o repositório remoto com `git push origin main`.
  • Verifique os remotos configurados no repositório local com o comando `git remote -v`.
  • Clonar um repositório remoto em seu computador com `git clone`.
  • Crie um branch de teste chamado "feature/teste" e faça alterações nele.
  • Compare as diferenças entre o branch "main" e o branch "feature/teste" usando `git diff`.
  • Crie um "tag" chamada "v1.0" no commit atual e visualize todas as tags com `git tag`.
  • Crie uma cópia exata do repositório atual em um diretório diferente com `git clone --bare`.
  • Renomeie o branch "main" para "master" com `git branch -m main master`.
