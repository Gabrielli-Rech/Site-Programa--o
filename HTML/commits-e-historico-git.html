O que são Commits e Histórico no Git?

Commits são como "marcadores" que registram mudanças em um repositório Git. Cada commit é um ponto no histórico do projeto e permite reverter ou examinar mudanças feitas ao longo do tempo. O histórico do Git ajuda a rastrear a evolução do código e colaborar com outras pessoas de forma eficiente.

Comando git commit

O comando git commit é usado para salvar as mudanças feitas no repositório. Cada commit é uma nova versão que inclui as alterações realizadas desde o último commit.

git add .  // Adiciona todas as mudanças para serem commitadas
git commit -m "Mensagem descritiva do commit"

O -m permite incluir uma mensagem descritiva, explicando as alterações feitas.

Comando git log

O comando git log exibe o histórico de commits em um repositório, mostrando informações como o hash do commit, autor, data e a mensagem de commit.

git log

Para visualizar um histórico mais compacto, você pode usar:

git log --oneline

Isso exibe cada commit em uma única linha, mostrando apenas o hash abreviado e a mensagem.

Comando git diff

O comando git diff permite comparar mudanças entre commits, branches ou a área de staging com a branch atual.

git diff  // Mostra as mudanças que ainda não foram commitadas
git diff HEAD  // Mostra as diferenças entre o último commit e a área de staging

Comando git show

O comando git show exibe detalhes sobre um commit específico, incluindo alterações feitas e a mensagem do commit.

git show

Substitua <hash-do-commit> pelo identificador do commit desejado para ver suas alterações detalhadas.

Comando git revert

O comando git revert é usado para criar um novo commit que desfaz as alterações feitas por um commit anterior.

git revert

Isso não exclui o commit original, mas cria um novo commit que reverte suas mudanças.

Comando git reset

O comando git reset é usado para desfazer commits. Ele pode ser usado de diferentes maneiras:

  • git reset --soft : Desfaz commits, mas mantém as mudanças na área de staging.
  • git reset --mixed : Desfaz commits e remove as mudanças da área de staging, mas as mantém no diretório de trabalho.
  • git reset --hard : Desfaz commits e remove as mudanças do diretório de trabalho, restaurando o estado do commit especificado.
