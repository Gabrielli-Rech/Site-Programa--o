Exercícios de Boas Práticas e Acessibilidade

Boas Práticas de CSS

  • Utilize variáveis CSS para cores e tamanhos em um projeto.
  • Adicione comentários explicativos no seu código CSS para maior compreensão.
  • Use unidades relativas (%, em, rem) em vez de fixas (px) para melhor responsividade.
  • Organize as regras CSS em um arquivo separado e mantenha uma estrutura lógica.
  • Minimize o uso de seletores muito específicos para facilitar a manutenção do código.
  • Utilize `flexbox` e `grid` para layouts em vez de `float` para melhor controle e flexibilidade.
  • Evite o uso excessivo de `!important`, pois isso dificulta a sobrescrição de regras.
  • Use `aria-label` e `aria-hidden` para tornar os elementos mais acessíveis.
  • Utilize o `box-sizing: border-box` para garantir que o padding e a borda não aumentem o tamanho dos elementos.
  • Utilize mixins e funções com pré-processadores CSS (Sass, Less) para evitar repetição de código.

Acessibilidade em CSS

  • Garanta contraste suficiente entre o texto e o fundo (pelo menos 4.5:1 para texto normal e 3:1 para texto grande).
  • Use `rem` para tamanhos de fonte para permitir a escalabilidade em leitores de tela.
  • Adicione `focus-visible` para fornecer um foco visível aos elementos interativos.
  • Evite usar apenas cores para transmitir informações (use texto ou ícones adicionais).
  • Certifique-se de que todos os elementos interativos tenham um `:hover` e `:focus` claramente visíveis.
  • Utilize `aria-hidden` para ocultar elementos que não devem ser lidos por leitores de tela.
  • Utilize `alt` nas imagens para fornecer uma descrição do conteúdo visual.
  • Teste seu site com ferramentas de acessibilidade como Lighthouse e WAVE.
  • Use um `label` com `for` em elementos de formulário para tornar os campos mais acessíveis.
  • Adicione `role="button"` e `tabindex` a elementos que não sejam botões mas que funcionam como botões.
