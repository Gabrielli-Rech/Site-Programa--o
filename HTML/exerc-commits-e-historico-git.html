Exercícios

  • 1. Criar um novo repositório Git e adicionar um arquivo de texto chamado README.md.
  • 2. Realizar o primeiro commit com uma mensagem "Initial commit".
  • 3. Modificar o arquivo README.md e fazer um commit com a mensagem "Atualizando o README".
  • 4. Verificar o histórico de commits usando o comando git log.
  • 5. Visualizar o histórico de commits em um formato compacto com git log --oneline.
  • 6. Mostrar o histórico de commits em formato de árvore com git log --graph.
  • 7. Realizar um commit com a opção --amend para modificar a última mensagem de commit.
  • 8. Listar os arquivos que foram modificados em um commit específico usando git show --name-only .
  • 9. Reverter um commit específico sem alterar o histórico com git revert .
  • 10. Resetar o repositório para um estado anterior com git reset --hard .
  • 11. Usar git diff para visualizar as mudanças entre o commit atual e a última versão com o repositório remoto.
  • 12. Criar um branch e realizar commits separados nele antes de fazer um merge com o branch principal.
  • 13. Usar git log --stat para ver o resumo das alterações feitas em um commit.
  • 14. Usar git rebase -i para reescrever o histórico de commits e combinar commits.
  • 15. Criar tags anotadas em um commit específico com git tag -a v1.0 .
  • 16. Visualizar os commits feitos por um autor específico com git log --author="nome do autor".
  • 17. Usar git log --since="2024-01-01" para mostrar commits feitos desde uma data específica.
  • 18. Buscar um commit específico usando git log -S "texto" para encontrar alterações que introduziram ou removeram uma palavra específica.
  • 19. Usar git blame para identificar quem fez a última modificação em cada linha de um arquivo.
  • 20. Comparar dois commits específicos para ver as diferenças usando git diff .
