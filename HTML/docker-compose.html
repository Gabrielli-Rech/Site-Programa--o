Exercícios Fáceis

  • 1. Crie um arquivo `docker-compose.yml` que inicie um container com o servidor Nginx.
  • 2. Configure um serviço MySQL com persistência de dados usando volumes.
  • 3. Adicione variáveis de ambiente em um serviço no `docker-compose.yml`.
  • 4. Configure um container Redis com configuração mínima.
  • 5. Execute múltiplos containers simultaneamente com o Docker Compose.
  • 6. Exponha uma porta específica para o serviço Nginx no `docker-compose.yml`.
  • 7. Adicione uma rede personalizada entre dois serviços no Docker Compose.
  • 8. Configure a reinicialização automática para um serviço no Docker Compose.
  • 9. Teste os comandos `docker-compose up` e `docker-compose down`.
  • 10. Use o comando `docker-compose ps` para verificar os serviços em execução.
  • 11. Crie um container MongoDB com autenticação básica configurada.
  • 12. Configure um volume para armazenar os dados de um banco PostgreSQL.
  • 13. Adicione um container Node.js para executar um script básico.
  • 14. Use um arquivo `.env` para armazenar variáveis de ambiente no Docker Compose.
  • 15. Crie um serviço que execute um script Python automaticamente.
  • 16. Configure um container para executar um servidor de cache Memcached.
  • 17. Adicione um container PHP com Apache usando o Docker Compose.
  • 18. Configure um servidor básico de mensagens com RabbitMQ.
  • 19. Crie um arquivo `docker-compose.yml` para executar um servidor FTP básico.
  • 20. Teste o comando `docker-compose logs` para visualizar os logs de serviços ativos.

  • 9. Adicione um serviço de administração, como Adminer ou PGAdmin, ao seu arquivo.

  • 12. Crie um ambiente com Python e Redis para executar tarefas em background.
  • 13. Configure múltiplos containers do mesmo serviço com diferentes portas.
  • 14. Use o Docker Compose para criar uma rede externa entre dois projetos diferentes.
  • 15. Configure um ambiente para executar o Prometheus com Grafana.
  • 16. Configure um ambiente para um serviço Flask com banco de dados PostgreSQL.

  • 20. Use o comando `docker-compose exec` para acessar um container em execução.
