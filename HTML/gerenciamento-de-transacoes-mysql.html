Exercícios Fáceis

  • Inicie uma transação com o comando `START TRANSACTION`.
  • Realize uma inserção de dados em uma tabela e confirme a transação com `COMMIT`.
  • Faça uma atualização de dados em uma tabela e desfaça a transação com `ROLLBACK`.
  • Verifique o status de uma transação com `SHOW ENGINE INNODB STATUS`.
  • Use o comando `SAVEPOINT` para criar um ponto de salvamento em uma transação.
  • Revert a transação para um ponto de salvamento específico com `ROLLBACK TO SAVEPOINT`.
  • Utilize o comando `SET autocommit = 0` para desabilitar o autocommit em uma sessão.
  • Verifique se o autocommit está ativado ou desativado com `SELECT @@autocommit`.
  • Inicie uma transação, faça uma inserção, e depois use `COMMIT` para salvar as mudanças.
  • Inicie uma transação, faça uma alteração e utilize `ROLLBACK` para reverter a mudança.

Exercícios Médios

  • Crie uma transação que insere dados em duas tabelas diferentes e use `COMMIT` para confirmar ambas as operações.
  • Teste a integridade dos dados em uma transação com uma verificação de condição antes do `COMMIT`.
  • Configure um `SAVEPOINT` em uma transação para reverter apenas uma parte da operação.
  • Simule um erro em uma transação e use `ROLLBACK` para desfazer todas as operações.
  • Utilize `SELECT FOR UPDATE` para bloquear registros em uma transação e impedir alterações de outros processos.
  • Crie uma transação aninhada com múltiplos `SAVEPOINT` e `ROLLBACK TO SAVEPOINT`.
  • Utilize a cláusula `LOCK TABLES` para garantir que os dados não sejam alterados por outras transações simultâneas.
  • Implemente uma transação para transferir dinheiro entre duas contas e utilize `ROLLBACK` em caso de erro.
  • Execute uma transação em um banco de dados com nível de isolamento `READ COMMITTED` e verifique seu efeito em consultas.
  • Crie um script que mostre como uma transação pode ser usada para evitar a condição de "dirty read".

Exercícios Difíceis

  • Implemente um sistema de gerenciamento de transações em que o banco de dados use `REPEATABLE READ` como nível de isolamento.
  • Desenvolva um processo para detectar e resolver deadlocks em uma transação.
  • Implemente uma transação com o nível de isolamento `SERIALIZABLE` e observe como ela afeta a concorrência.
  • Utilize transações para gerenciar operações de inserção em massa com controle de erros em tempo real.
  • Simule uma falha de sistema e mostre como a transação pode ser recuperada com `ROLLBACK` e um ponto de recuperação.
  • Crie um script de transação com múltiplas tabelas e garanta a integridade referencial com transações aninhadas.
  • Use o `SET TRANSACTION` para definir propriedades específicas para uma transação, como nível de isolamento e tempo de espera.
  • Implemente uma estratégia de transações usando a cláusula `WAITFOR` para controlar o tempo de espera em transações bloqueadas.
  • Teste a manipulação de transações em um banco de dados distribuído e verifique como as operações são coordenadas.
  • Crie um sistema de transação com um `BEFORE COMMIT` trigger para validar mudanças antes da confirmação final.

Desafios

  • Implemente uma transação que utiliza múltiplos pontos de salvamento para dividir operações complexas em etapas menores.
  • Crie um mecanismo de controle de transações para um sistema de inventário que tenha múltiplos fornecedores e controle de estoque.
  • Desenvolva uma transação que faça uso de um `AFTER COMMIT` trigger para enviar notificações após a confirmação de dados.
  • Implemente uma solução para detectar e gerenciar transações paralelas em um sistema de banco de dados com alto volume de operações.
  • Crie um sistema de relatórios que utilize transações para garantir que as consultas sejam consistentes em um banco de dados em constante mudança.
  • Configure um banco de dados para registrar o histórico de transações em uma tabela de auditoria com `COMMIT` e `ROLLBACK`.
  • Desenvolva um script de transação para executar uma análise de impacto em massa antes de atualizar um conjunto de registros.
  • Implemente um mecanismo de controle de transações com uso de `LOCK TABLES` para manter a integridade em cenários de alto acesso simultâneo.
  • Configure transações para automatizar a criação de backups de dados em um sistema de alta disponibilidade.
  • Desenvolva uma lógica de transação com `SAVEPOINT` para realizar um "rollback parcial" em um processo de atualização de registros.
