Exercícios de Instalação e Configuração

  • Instale o Docker em sua máquina seguindo a documentação oficial do Docker para seu sistema operacional (Linux, Windows, macOS).
  • Verifique se o Docker foi instalado corretamente executando o comando `docker --version` no terminal.
  • Inicie o serviço Docker e verifique seu status com `systemctl status docker` (Linux) ou pelo painel de controle do Docker Desktop (Windows/macOS).
  • Crie um usuário local e adicione-o ao grupo `docker` para executar comandos Docker sem sudo no Linux.
  • Configure o Docker para iniciar automaticamente após o boot do sistema usando `systemctl enable docker` (Linux).
  • Configure a interface do Docker Desktop em um ambiente Windows ou macOS para utilizar a virtualização de containers com recursos específicos.
  • Instale o Docker Compose e verifique a versão com `docker-compose --version`.
  • Crie um arquivo `docker-compose.yml` simples que defina um serviço de webserver (exemplo: Nginx).
  • Execute um container Nginx em modo interativo e acesse a página de boas-vindas do Nginx pelo navegador.
  • Configure variáveis de ambiente no Docker Compose e passe parâmetros de configuração para um container.
  • Utilize volumes no Docker Compose para persistir dados de um container (exemplo: um banco de dados).
  • Configura o Docker para usar um proxy HTTP/HTTPS em caso de uma rede corporativa ou firewall.
  • Configure o Docker para usar um registro de imagens privado, configurando o `~/.docker/config.json` com credenciais.
  • Desinstale o Docker em um sistema Linux e remova todos os containers e imagens associadas.
  • Utilize o Docker CLI para criar e rodar containers em diferentes ambientes (produção, desenvolvimento, etc.).
  • Crie um Dockerfile básico para um app de "Hello World" em Node.js e construa uma imagem a partir dele.
  • Explore o comando `docker stats` para monitorar o uso de recursos de containers em tempo real.
  • Configure o `docker-compose.yml` para iniciar múltiplos serviços interdependentes (exemplo: frontend e backend).
  • Habilite logs detalhados de um container para depuração usando `docker logs `.
  • Experimente a configuração de limites de CPU e memória para containers em um `docker-compose.yml` usando as opções `mem_limit` e `cpus`.
  • Configure um container Docker para rodar em modo de reinício automático com a política `restart: always`.
