Exercícios Práticos

  • Crie duas tabelas: clientes e pedidos. Adicione um relacionamento de um para muitos, onde cada cliente pode ter vários pedidos.
  • Crie uma tabela de produtos e uma tabela de categorias. Estabeleça um relacionamento de muitos para um entre os produtos e suas categorias.
  • Desenvolva um esquema de banco de dados com uma tabela de autores e outra de livros, onde um autor pode ter vários livros. Adicione uma chave estrangeira em livros para referenciar autores.
  • Crie um banco de dados com tabelas estudantes e disciplinas e uma tabela intermediária chamada matriculas para representar a relação de muitos para muitos.
  • Implemente um esquema com uma tabela funcionarios e uma tabela departamentos, onde um funcionário pertence a um departamento. Adicione uma chave estrangeira em funcionarios para referenciar departamentos.
  • Crie um relacionamento de um para um entre as tabelas pessoas e detalhes_pessoais, onde cada pessoa tem um conjunto único de detalhes pessoais.
  • Crie uma tabela de clientes e uma tabela de enderecos, onde cada cliente pode ter vários endereços. Adicione uma chave estrangeira em endereco que se refira ao cliente.
  • Faça uma tabela de cursos e outra de alunos com um relacionamento de muitos para muitos. Crie uma tabela intermediária chamada aluno_curso.
  • Crie um banco de dados com tabelas pedidos e itens_pedido, onde cada pedido pode ter vários itens. Adicione uma chave estrangeira em itens_pedido para se referir a pedidos.
  • Crie uma tabela de categorias e uma tabela de subcategorias, onde uma subcategoria pertence a uma categoria pai. Use uma chave estrangeira em subcategorias para referenciar categorias.
