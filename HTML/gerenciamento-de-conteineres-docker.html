Exercícios Fáceis

  • Liste todos os contêineres em execução com o comando `docker ps`.
  • Liste todos os contêineres, incluindo os que estão parados, com `docker ps -a`.
  • Verifique a versão do Docker instalada com o comando `docker --version`.
  • Crie um contêiner simples do Nginx com o comando `docker run`.
  • Visualize os logs de um contêiner com o comando `docker logs `.
  • Verifique a utilização de recursos (CPU, memória) de um contêiner com `docker stats`.
  • Execute um comando dentro de um contêiner em execução com `docker exec`.
  • Nomeie um contêiner durante sua criação com `docker run --name `.
  • Exiba as informações detalhadas de um contêiner com `docker inspect `.
  • Remova um contêiner com o comando `docker rm `.

Exercícios Médios

  • Crie e execute um contêiner com um volume persistente montado.
  • Crie um contêiner em modo interativo e execute comandos no shell do contêiner.
  • Parar e iniciar um contêiner existente com `docker stop` e `docker start`.
  • Reinicie um contêiner com o comando `docker restart`.
  • Copie arquivos de e para um contêiner com `docker cp`.
  • Use `docker commit` para criar uma nova imagem a partir de um contêiner existente.
  • Defina variáveis de ambiente ao criar um contêiner com `docker run -e`.
  • Conecte-se a um contêiner em execução com `docker attach`.
  • Crie um contêiner que expose uma porta específica para a máquina host.
  • Crie e execute um contêiner com uma imagem específica e faça alterações na configuração do contêiner.

Exercícios Difíceis

  • Configure uma rede personalizada para contêineres com `docker network create` e conecte contêineres a ela.
  • Crie e gerencie contêineres usando Docker Compose.
  • Use `docker inspect` para verificar as configurações de rede de um contêiner.
  • Limpe todos os contêineres parados e imagens não utilizadas com `docker system prune`.
  • Construa uma imagem Docker com um `Dockerfile` e um comando `docker build`.
  • Configure a escalabilidade de contêineres usando o comando `docker service create`.
  • Automatize a criação de contêineres e deploys com scripts de shell usando `docker run`.
  • Use o comando `docker exec` para realizar operações em múltiplos contêineres simultaneamente.
  • Monitore logs em tempo real com o comando `docker logs -f `.
  • Use `docker update` para alterar as configurações de um contêiner em execução.

Desafios

  • Crie um contêiner Docker que execute uma aplicação web com suporte a SSL/TLS.
  • Monte um sistema de backup e restauração de contêineres usando `docker export` e `docker import`.
  • Implemente uma política de reinício de contêineres para garantir que eles sejam automaticamente reiniciados após falhas.
  • Crie e execute um contêiner Docker que faça a sincronização de arquivos com um serviço de armazenamento em nuvem.
  • Implemente um script que crie um contêiner de banco de dados e conecte outro contêiner a ele como cliente.
  • Configure um contêiner com uma ferramenta de monitoramento (como Prometheus) e visualize métricas de desempenho.
  • Desenvolva um contêiner com suporte a CI/CD integrando com GitHub Actions ou GitLab CI/CD.
  • Crie uma imagem Docker personalizada que contenha uma aplicação em Node.js com dependências específicas.
  • Configure contêineres de serviços em um cluster de Docker Swarm para implementar alta disponibilidade.
  • Use volumes para compartilhar arquivos entre o host e contêineres e permita a edição em tempo real.
