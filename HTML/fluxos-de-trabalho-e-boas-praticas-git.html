Exercícios Fáceis

  • Crie um repositório local e faça seu primeiro commit.
  • Clone um repositório remoto para a sua máquina local.
  • Faça um git pull para atualizar a branch local com as alterações remotas.
  • Comite mudanças em um arquivo com uma mensagem de commit adequada.
  • Liste todos os branches locais e remotos com git branch.
  • Verifique o histórico de commits com git log.
  • Crie e alterne para uma nova branch com git checkout -b.
  • Faça git status para verificar as mudanças não comitadas.
  • Adicione arquivos ao stage com git add.
  • Remova um arquivo do stage com git reset.

Exercícios Médios

  • Resolva conflitos de merge e faça um commit de merge.
  • Revise um commit específico usando git revert.
  • Faça rebase de uma branch com git rebase.
  • Renomeie uma branch com git branch -m.
  • Crie um arquivo .gitignore e adicione arquivos que devem ser ignorados.
  • Use tags para marcar uma versão específica do projeto.
  • Compare dois commits com git diff.
  • Realize uma fusão de branches com git merge.
  • Use git stash para salvar temporariamente mudanças não comitadas.
  • Use git cherry-pick para aplicar commits de outra branch.

Exercícios Difíceis

  • Configure e use hooks do Git para automação.
  • Reescreva o histórico de commits com git rebase -i.
  • Use git filter-branch para reescrever o histórico de um repositório.
  • Resolva conflitos de rebase e continue com git rebase --continue.
  • Crie e use aliases personalizados com git config.
  • Manipule branches remotas com git push --force.
  • Use git reflog para encontrar commits perdidos.
  • Configure um repositório remoto para usar SSH em vez de HTTPS.
  • Integre Git com uma ferramenta de CI/CD (como GitHub Actions ou GitLab CI/CD).
  • Realize uma operação git bisect para encontrar um commit que introduziu um erro.

Desafios

  • Crie um fluxo de trabalho de integração contínua com GitHub Actions.
  • Configure um repositório com git subtree para gerenciar submódulos.
  • Configure um repositório para suportar trabalho em equipe usando git pull --rebase.
  • Use git submodule para incluir repositórios externos.
  • Implemente um fluxo de GitFlow em um projeto com múltiplas branches.
  • Reescreva commits de forma interativa para manter um histórico limpo.
  • Use git merge --squash para combinar commits em um só antes de fazer merge.
  • Crie e gerencie pull requests de forma eficaz em um repositório compartilhado.
  • Configure permissões de acesso e gerenciamento de branches em um repositório remoto.
  • Implemente políticas de revisão de código e merge request no GitHub/GitLab.
