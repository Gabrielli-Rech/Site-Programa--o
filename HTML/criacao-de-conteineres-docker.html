O que são Contêineres Docker?

Contêineres Docker são unidades leves de software que empacotam código e suas dependências para garantir que a aplicação seja executada de forma consistente em qualquer ambiente. Eles compartilham o mesmo sistema operacional, mas são isolados uns dos outros.

Criação de Contêineres

Para criar e executar um contêiner Docker, você pode usar o comando docker run. Este comando baixa a imagem especificada (caso não esteja local) e executa um contêiner com base nessa imagem.

Comando básico:

docker run nome_da_imagem

Exemplo de criação de um contêiner a partir da imagem oficial do Ubuntu:

docker run -it ubuntu

A opção -it permite que você interaja com o contêiner em modo de terminal.

Comandos Úteis para Contêineres

Alguns comandos úteis para gerenciar contêineres:

  • Listar contêineres em execução: docker ps
  • Listar todos os contêineres (ativos e inativos): docker ps -a
  • Parar um contêiner: docker stop
  • Iniciar um contêiner parado: docker start
  • Remover um contêiner: docker rm

Executando Contêineres em Segundo Plano

Para executar um contêiner em segundo plano, use a opção -d (detached mode):

docker run -d nome_da_imagem

Isso permite que o contêiner seja executado em background, liberando o terminal para outros comandos.

Montando Volumes e Expondo Portas

Você pode montar volumes e expor portas para interagir com o contêiner:

docker run -d -p 8080:80 -v /caminho/no/host:/caminho/no/container nome_da_imagem

Exemplo de uso:

docker run -d -p 3000:3000 -v /meu/app:/app node:14

Isso mapeia a porta 3000 do contêiner para a porta 3000 do host e monta o diretório local /meu/app dentro do contêiner em /app.
